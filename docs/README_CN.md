# Mystery Analysis Skill

一个专门用于分析推理小说和侦探故事的 Claude Code 技能包。

## 功能特性

### 核心能力

- **文本导入**: 支持 Markdown、纯文本、PDF，以及通过脚本提取的 EPUB 正文
- **章节解析**: 自动识别和分段章节结构
- **结构化分析**: 提取细节、异常、伏笔和洞察
- **一致性检查**: 检测章节间的矛盾和时间线问题
- **关系图谱**: 生成角色和实体关系网络
- **Vault 创建**: 自动创建完整的 Obsidian vault 仓库
- **多格式导出**: 支持 JSON、Markdown、Obsidian 格式

### 分析维度

1. **细节 (Details)**: 文本中的具体观察和事实
2. **异常 (Anomalies)**: 可疑或不寻常的元素
3. **伏笔 (Foreshadowing)**: 可能的情节铺垫
4. **洞察 (Insights)**: 基于证据的解读

## 使用方法

### 基础分析

```
用户: "分析这个推理小说第 5 章的线索和伏笔"

Claude: [调用推理分析技能]
- 解析章节内容
- 提取细节、异常、伏笔和洞察
- 返回结构化分析结果
```

### 一致性检查

```
用户: "检查第 3-7 章中嫌疑人不在场证明的矛盾"

Claude: [调用推理分析技能]
- 加载指定章节范围
- 比较陈述和时间线
- 识别不一致之处
- 报告矛盾点及章节引用
```

### 关系图谱

```
用户: "生成第 1-10 章的角色关系图"

Claude: [调用推理分析技能]
- 提取角色名称和关系
- 识别互动模式
- 创建图谱结构
- 返回可视化数据
```

### 创建 Obsidian Vault

```
用户: "将分析结果导出到 Obsidian vault"

Claude: [调用推理分析技能]
- 询问 vault 创建位置
- 创建完整的文件夹结构
- 生成书籍信息页、章节分析、角色卡片
- 使用双链 [[]] 连接所有实体
- 添加标签和元数据
- 返回 vault 路径供 Obsidian 打开
```

## 输出格式

```json
{
  "details": ["文本中的具体观察"],
  "anomalies": ["可疑或不寻常的元素"],
  "foreshadowing": ["可能的伏笔设置"],
  "insights": ["基于证据的解读"],
  "summary": "章节简要概述",
  "entities": ["角色、物品、地点"],
  "chapters": ["章节引用"]
}
```

## 设计原则

### 证据优先

- 所有分析必须基于文本中的明确证据
- 避免基于先验知识的推测
- 优先观察而非解读
- 为重要主张提供来源证据

### 语言保持

- 自动匹配源文本语言
- 中文源文本 → 中文分析
- 英文源文本 → 英文分析
- 避免分析过程中的语言切换

### 细读增强

- 帮助读者注意容易忽略的细节
- 识别跨章节的模式和联系
- 追踪值得关注的开放性问题
- 增强而非替代细读体验

## 质量标准

### 应包含

✓ 具体引用或转述
✓ 具体细节（姓名、时间、地点、行为）
✓ 可观察的角色行为
✓ 明确的对话和反应
✓ 每个主张的文本证据

### 应避免

✗ 泛泛的情节总结
✗ 没有文本支持的推测
✗ 基于故事结局的先验知识
✗ 语言切换
✗ 替读者解答谜题

## 使用场景

### 推理小说读者

- 追踪长篇小说中的线索
- 识别模式和联系
- 首次阅读时注意伏笔
- 准备读书会讨论

### 作家

- 分析推理小说的结构和节奏
- 学习如何埋设线索
- 研究伏笔技巧
- 理解一致性要求

### 研究者

- 进行叙事分析
- 研究侦探小说惯例
- 分析情节构建
- 比较推理写作风格

## 技术支持

### 支持的格式

- Markdown (.md)
- 纯文本 (.txt)
- PDF (.pdf)
- EPUB (.epub，需先运行 `python scripts/extract_epub.py <文件>.epub --output extracted.json`)

### EPUB 使用说明

- 不要把 `.epub` 二进制文件直接当成已读取正文。
- 先运行提取脚本，再根据提取出的章节文本分析。
- 如果脚本没有提取出可读章节，应该停止分析并明确说明失败，而不是根据书名或先验知识作答。

### 模型兼容性

- Claude (Anthropic)
- DeepSeek
- OpenAI GPT
- 其他兼容的 LLM

### 输出格式

- JSON 结构化数据
- Markdown 笔记
- Obsidian 知识库
- 图谱可视化数据

## Obsidian Vault 结构

当创建 Obsidian vault 时,会生成以下结构:

```
mystery-vault-[书名]/
├── Books/              # 书籍分析
│   └── [书名]/
│       ├── 00-书籍信息.md
│       ├── 01-第一章分析.md
│       ├── 02-第二章分析.md
│       └── ...
├── Analysis/           # 跨章节分析
│   ├── 线索追踪.md
│   ├── 一致性检查.md
│   └── 伏笔汇总.md
├── Characters/         # 角色卡片
│   ├── 角色A.md
│   ├── 角色B.md
│   └── ...
├── Templates/          # 模板文件
│   ├── 章节分析模板.md
│   └── 角色卡片模板.md
└── Home.md            # 主索引页
```

### Vault 特性

- 双链系统: 使用 `[[角色名]]` `[[物品]]` 连接所有实体
- 标签分类: `#mystery` `#chapter` `#character` 等标签
- 待办追踪: `- [ ]` 追踪未解之谜
- 时间线: 记录事件发生顺序
- 关系网络: 角色和实体的连接关系

### 使用 Obsidian 打开

1. 打开 Obsidian 应用
2. 选择 "打开文件夹作为仓库"
3. 选择生成的 vault 路径
4. 开始浏览和编辑分析笔记
